Обновление системы на автомате при помощи chmod


В этой статье делюсь личным опытом использования команды chmod. Этот способ работает на всех дистрибутивах Linux.

Что такое chmod?

сhmod или change mode (сменить режим) - это утилита для изменения прав доступа к файлам и директориям.

Обновление Ubuntu на автомате.

Когда поставил себе Ubuntu вообще не понимал как её обновлять. Разобравшись, чувствовал себя хакером). Мне нравилось вбивать всё вручную. Только вот не понимал зачем копировать, когда быстрее напечатать самому?
Со временем, вся эта рутина просто достала. Короче, пришло время упростить процесс обновления системы.
Пошагово опишу как это делаю я. Рекомендую посмотреть видео, что бы увидеть как это выглядит в реальности. Так же это даст более ясную картину происходящиго.

    Инструкция:

  • Шаг 0:
    • Для начала нужно создать директорию "update_system" в которой будет храниться файл с обновлением системы *:
    mkdir update_system
    * Папку лучше создать в "домашней папке", а не на "рабочем столе".

  • Шаг 1:
    • Далее заходим в папку "update_system":
    cd update_system
    И создаем там файл. Назовем его к примеру "update.run":
    touch update.run

  • Шаг 2:
    • В документ "update.run" записать эти команды:
    sudo apt update && sudo apt upgrade -y && sudo apt autoclean -y && sudo apt autoremove -y
    "sudo" - права суперпользователя.
    "apt"- пакетный менеджер.
    "update"- обновить список пакетов.
    "upgrade"- обновить установленные пакеты.
    "autoclean"- удалить неиспользуемые пакеты из кэша.
    "autoremove"- удаление ненужных зависимостей.
    "-y"- ключ "yes" согласиться на все действия.
    "&&"- логическое "И" позволяет последовательно выполнять команды.
  • Шаг 3:
    • Теперь нужно задать параметры для файла "update.run" при помощи chmod:
    chmod +x update.run

  • Шаг 4:
    • Теперь можно запустить файл на исполнение:
    ./update.run
    Вводим пароль суперпользователя.
    Одна команда заменила кучу операций!

    Чтобы не заходить в папку при помощи "cd", делайте так:
    ./update_system/update.run
